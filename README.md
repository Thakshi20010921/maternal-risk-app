# 🩺 Maternal Health Risk Predictor

An interactive web app that predicts maternal health risk level (**Low** / **High**)
from patient vitals and medical history, built from a data science project comparing
5 machine learning classifiers.

**Live demo:** _add your Streamlit Cloud link here after deploying_

---

## What it does

- **Predict Risk** — enter a patient's vitals (age, blood pressure, blood sugar,
  BMI, heart rate, etc.) and get an instant risk prediction with confidence scores
  and a visual gauge.
- **Model Insights** — compare performance (accuracy, precision, recall, F1, ROC AUC)
  across 5 trained models: Logistic Regression, Decision Tree, Random Forest,
  tuned KNN, and tuned SVM. Includes confusion matrices, feature importance, and
  dataset exploration charts.
- **About** — project background and feature reference.

## Tech stack

- **Python** — model training and app logic
- **scikit-learn** — Logistic Regression, Decision Tree, Random Forest, KNN, SVM
- **Streamlit** — web interface
- **Plotly** — interactive charts (gauges, bar charts, confusion matrices, heatmaps)
- **pandas / numpy** — data processing

## Project structure

```
maternal-risk-app/
├── app.py                 # Streamlit app (UI + prediction logic)
├── train_model.py         # Trains all 5 models, picks the best, saves artifacts
├── requirements.txt       # Python dependencies
├── data/
│   └── maternal_health_data.csv
├── models/
│   ├── best_model.pkl      # Generated by train_model.py
│   ├── scaler.pkl          # Generated by train_model.py
│   └── metrics.json        # Generated by train_model.py
└── .streamlit/
    └── config.toml         # App theme
```

## Run locally

```bash
# 1. Clone the repo
git clone <your-repo-url>
cd maternal-risk-app

# 2. Install dependencies
pip install -r requirements.txt

# 3. Train the models (generates models/*.pkl and metrics.json)
python train_model.py

# 4. Launch the app
streamlit run app.py
```

The app will open at `http://localhost:8501`.

## Deploy to Streamlit Community Cloud (free, public link)

1. Push this folder to a **public GitHub repository**.
2. Make sure `models/best_model.pkl`, `models/scaler.pkl`, and `models/metrics.json`
   are committed (run `train_model.py` locally first and commit the generated files —
   this avoids retraining on every app restart).
3. Go to [share.streamlit.io](https://share.streamlit.io) and sign in with GitHub.
4. Click **"New app"**, select your repo, branch, and set the main file path to `app.py`.
5. Click **Deploy**. You'll get a public URL like:
   `https://your-app-name.streamlit.app`
6. Share that link on LinkedIn 🎉

> **Tip:** If you'd rather not commit the trained model files, you can instead add a
> small startup check in `app.py` that runs `train_model.py` automatically if
> `models/best_model.pkl` doesn't exist yet. Committing the artifacts is simpler and
> faster for users, though.

## Model performance summary

| Model | Accuracy | F1 Score |
|---|---|---|
| Random Forest | ~0.987 | ~0.984 |
| SVM (tuned) | ~0.979 | ~0.973 |
| KNN (tuned) | ~0.974 | ~0.968 |
| Logistic Regression | ~0.970 | ~0.963 |
| Decision Tree | ~0.953 | ~0.941 |

*(Exact values are recalculated each time `train_model.py` runs; see the in-app
Model Insights page for live numbers.)*
